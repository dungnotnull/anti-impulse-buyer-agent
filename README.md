A browser extension + local AI agent that intercepts purchases at the **exact moment** you click "Buy Now". It freezes the transaction, delivers a cold financial analysis personalized to your spending history, and requires you to solve a logic challenge before the purchase can proceed.

Built on behavioral science research — the 5-minute mandatory cooling-off period forces a psychological shift from emotional (System 1) to rational (System 2) decision-making, breaking the manipulation loop engineered by modern e-commerce dark patterns.

---

## The Problem

| Statistic | Source |
|-----------|--------|
| **40–80%** of all purchases are unplanned or impulse-driven | Journal of Consumer Psychology, 2023 |
| **$5,400/year** wasted on impulse purchases by the average US consumer | Slickdeals, 2022 |
| **3× more** impulse buying online vs. physical stores | MIT Sloan, 2021 |
| **34% higher return rates** for late-night purchases (10PM–2AM) | Nielsen, 2023 |

Current solutions — budgeting apps, card spending limits — operate **after the damage is done**. No tool intervenes at the critical moment of purchase with personalized, contextual financial friction. This is the gap anti-impulse-buyer fills.

## Features

### Core
- **🚫 Checkout Interception** — Automatically detects buy-now/checkout buttons on e-commerce sites and intercepts the click
- **⏱️ Mandatory Freeze** — 2–5 minute countdown timer based on your impulsivity score
- **🧠 Financial Narratives** — Personalized "cold reality check" generated by AI (Claude, GPT-4, or local Ollama)
- **🧩 Logic Challenge Gate** — Solve a puzzle before you can proceed. Difficulty scales with impulsivity (easy → medium → hard)
- **📊 Spending Dashboard** — React popup with charts: decision breakdown, score trends, money saved, streak tracker
- **✅ Essential Items Whitelist** — Mark recurring essentials (groceries, medicine) that bypass the freeze

### AI / ML
- **🔬 Product Classification** — Auto-categorizes products (fashion, electronics, groceries, etc.) via BART-MNLI zero-shot. Falls back to keyword matching if model not installed.
- **🔗 Semantic Essential Matching** — Computes cosine similarity between products and your essential items list via all-MiniLM-L6-v2. Falls back to Jaccard overlap.
- **❤️ Sentiment Context** — Detects emotional state from browsing session text via Twitter-RoBERTa. Amplifies impulsivity score when negative emotions are detected.
- **🧮 Composite Score** — 5-factor impulsivity algorithm: time-of-day + essential-item deviation + recent purchase frequency + category weight + price-to-income ratio, multiplied by sentiment

### Dark Pattern Detection
- **🔍 Real-time Scanner** — Scans page DOM for dark patterns: fake scarcity ("Only 2 left!"), false urgency (countdown timers), social proof manipulation, confirmshaming, subscription traps
- **🏷️ Visual Highlighting** — Color-coded badges on detected manipulative elements
- **📋 Server-side Analysis** — Additional NLP analysis of page text sent via intercept endpoint

### Privacy & Data
- **🔒 Local-First** — All spending history processed on your machine. No account required.
- **🔐 AES-256 Encryption** — Optional SQLite database encryption via sqlcipher3
- **📥 CSV Export** — Export all spending events and wish list data
- **🗑️ One-Click Purge** — Delete all data instantly
- **☁️ Cloud is Opt-In** — External LLM APIs (Claude/GPT-4) only activate when you provide API keys

### Self-Improving Knowledge Loop
- **🤖 Automated Research** — Weekly crawler searches ArXiv, Semantic Scholar, and HuggingFace Papers for new behavioral economics research and ML models
- **📝 Auto-Append** — New findings are automatically added to the research knowledge base
- **📊 Model Evaluation** — Benchmarks new HuggingFace models against a 30-product test set. Flags any model >3% better than current for manual review
- **🔄 Prompt Optimization** — Analyzes user decision feedback to suggest LLM prompt refinements

## Quick Start

### Prerequisites
- **Python 3.11+** with `pip`
- **Node.js 18+** with `npm` (for building the popup dashboard)
- **Chrome** or **Firefox** (for loading the unpacked extension)

### Setup

**Windows:**
```batch
setup.bat
```

**macOS/Linux:**
```bash
chmod +x setup.sh && ./setup.sh
```

**Or manually:**
```bash
# 1. Create virtual environment & install Python deps
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 2. Build the React popup dashboard
cd extension/popup
npm install && npx vite build
cd ../..

# 3. Start the backend
python -m backend.main
```

### Load the Extension

**Chrome:**
1. Go to `chrome://extensions`
2. Enable **Developer Mode** (toggle top-right)
3. Click **Load unpacked**
4. Select the `extension/` folder

**Firefox:**
1. Go to `about:debugging#/runtime/this-firefox`
2. Click **Load Temporary Add-on**
3. Select `extension/manifest.json`

### First Run
1. After loading the extension, click its icon in the toolbar
2. Open the **Settings** tab and enter your monthly income for salary-calibrated narratives
3. Add essential items in the **Essentials** tab (groceries, bills, etc. that should bypass the freeze)
4. Visit any supported e-commerce site and click "Buy Now" — the overlay will appear!

| Layer | Technology | Role |
|-------|-----------|------|

| **Content Scripts** | Vanilla JavaScript (ES2022) | Button scanning, metadata extraction |

| **Local Backend** | Python 3.12 + FastAPI | API server, scoring, LLM routing |
| **Database** | SQLite + SQLAlchemy + optional sqlcipher3 | Spending history, essentials, profiles |
| **Product Classifier** | `facebook/bart-large-mnli` (zero-shot) → keyword fallback | Category classification |
| **Embedding Matcher** | `sentence-transformers/all-MiniLM-L6-v2` → Jaccard fallback | Essential items similarity |
| **Sentiment** | `cardiffnlp/twitter-roberta-base-sentiment-latest` → neutral fallback | Emotional context multiplier |
| **LLM Router** | Claude API → OpenAI → Ollama → deterministic template | Financial narrative generation |
| **Task Queue** | Celery + Redis (optional, graceful fallback) | Async ML inference |

## API Reference

All endpoints accept and return JSON. Base URL: `http://localhost:7432`

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Backend health check |

### Impulsivity Score Formula

```
ImpulsivityScore = (
    0.25 × time_of_day_factor        # 0.0 (noon) → 1.0 (1 AM)
  + 0.20 × essential_similarity_inv  # 1 - cosine(product, essentials_list)
  + 0.20 × recent_purchase_frequency # purchases last 48h / baseline
  + 0.20 × price_to_income_ratio     # item_price / (monthly_avg / 30)
  + 0.15 × category_impulse_weight   # fashion=0.9, groceries=0.2, etc.
) × sentiment_multiplier             # 1.0 (neutral) → 1.3 (negative)
```

| Score Range | Tier | Freeze Duration | Challenge Difficulty | Action |
|-------------|------|-----------------|---------------------|--------|
| 0.0–0.4 | Low | 2 minutes | Easy (arithmetic) | Educational nudge |
| 0.4–0.7 | Medium | 5 minutes | Medium (pattern) | Full intervention |
| 0.7–1.0 | High | 5 minutes | Hard (word problem) | Full intervention + wish list suggestion |

## Research & Behavioral Science

This project is grounded in established behavioral economics research. The knowledge base (`SECOND-KNOWLEDGE-BRAIN.md`) contains:

- **Dual-Process Theory** (Kahneman) — The 5-minute freeze forces System 2 thinking
- **Ego Depletion** (Baumeister) — Late-night weighting accounts for depleted willpower
- **Scarcity Heuristic** — Dark pattern detector specifically flags scarcity messaging
- **Present Bias** — Financial narratives translate price into days of work
- **Implementation Intentions** — Essential items list + wish list quarantine operationalize pre-commitment
- **Regret Anticipation** — Logic challenge includes financial reflection prompts

## Configuring Cloud AI (Optional)

For richer financial narratives, set these environment variables before starting the backend:

```bash
# Windows (Command Prompt)
set CLAUDE_API_KEY=sk-ant-...
set OPENAI_API_KEY=sk-proj-...
python -m backend.main

# macOS/Linux
CLAUDE_API_KEY=sk-ant-... OPENAI_API_KEY=sk-proj-... python -m backend.main
```

Or set them permanently in `backend/config.py` or a `.env` file.

The LLM router tries backends in priority order: `ollama → claude → openai → template`. If all fail, it falls back to a deterministic template. The extension remains fully functional with no API keys — you just get the template narratives instead of AI-generated ones.

## Privacy

| Concern | Mitigation |
|---------|-----------|
| **Spending data** | Stored locally in SQLite. Optional AES-256 encryption via sqlcipher3 |
| **Product data sent to LLM** | Only if you explicitly enable cloud AI and provide API keys. Default is local-only |
| **Browser permissions** | Minimal: `activeTab`, `storage`, `scripting`, `alarms`. No broad host permissions |
| **Network exposure** | Backend listens on `127.0.0.1:7432` only. CORS restricted to extension origin |
| **Data retention** | User-configurable (30/90/365 days). One-click purge always available |
| **No account required** | Extension works fully offline and account-free |
